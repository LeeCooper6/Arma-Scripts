Welcome

Welcome to my repository of open source scripts for Arma!

I've specifically designed these scripts to be as user-friendly to non-technical (non-programmer) players who just want things to work but don't have the time or knowledge to learn scripting and code.

What Are Scripts?

Scripts are little files of code run in-game that to do all sorts of wonderful and amazing things - or horrible things, depending on how you set it up (with great power comes great responsibility and all that).

Scripts in Arma are typically either .sqs or .sqf files which can do any number of things, though there are also some more niche kinds of scripts like FSMs that define how AI units behave.

However, the files themselves matter less than the actual code they contain, which can enable you to do all kinds of things you are otherwise totally unable to do in Arma.

How Do I Use Scripts?

1. Download the Script

Firstly, you have to download the script you wish to use. Alternatively, you could make your own script, copy the code from my script, and paste it in yours.

2. Place the Script in Your Mission Folder

Secondly, you need to place the script in the mission folder of the mission you want to use it in, which will probably be in

C:\Users\YourUserName\Documents\Arma 3 - Other Profiles\YourProfileName\missions\YourMissionName

on Windows.

3. Run the Script

Thirdly, you need to execute the script by running code like this:

scriptDone = [someArguments] execVM "someScript.sqf";

scriptDone is a variable used to determine whether or not the script has completed running. Almost always, you only have to write null for that and never think about it again.

someArguments is an array of parameters or settings you need to send to the script so it knows how to work and what to do. Arguments allow you to customize how the script works and what it does. However, every script is different and requires different arguments. Some require very complex arrays of arguments, others require none at all. All my scripts here come with thorough documentation explaining precisely what you need to write to make them work.

execVM is a scripting command to execute .sqf scripts, while exec is a command to execute .sqs scripts. Typically, you'll use execVM to run .sqf scripts, which have the unique and powerful ability to wait for something to happen before moving on.

someScript.sqs or someScript.sqf is the name of the script. Again, the script has to be in the mission folder of the mission you want to use it in.

Again, all my scripts here come with thorough documentation explaining precisely what you need to write to make them work, so don't worry about it becoming too complicated, I'll make sure it's simple and easy.

Example

Let's say you want to enable the player to jump - something you can't do in vanilla Arma. Here's what you'd need to do:

  1. Download my EnableJump.sqf script.
  2. Put the EnableJump.sqf script in the mission folder of the mission you want to be able to jump in.
  3. Paste null = execVM "EnableJump.sqf"; in the player's Init box (or anywhere, actually).

It works! It's that easy.
